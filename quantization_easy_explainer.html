크레용 비유로 이해하기
원래 이미지는 256가지 색(0~255)을 쓸 수 있는 고급 물감 세트예요.
양자화는 이걸 더 적은 색의 크레용 세트로 줄이는 거예요.
크레용 수가 적을수록 표현이 뭉툭해지고 계단 현상이 생겨요!
아래 슬라이더로 직접 확인해보세요 — 단계가 줄어들수록 사진이 거칠어집니다.

픽셀로 보는 양자화 — 숫자가 어떻게 바뀌나
예시: 4단계(levels=4)로 양자화할 때, 각 픽셀 값이 어떤 그룹에 속하는지 보세요.
그룹 0 (0~63)
→ 픽셀값 0
그룹 1 (64~127)
→ 픽셀값 85
그룹 2 (128~191)
→ 픽셀값 170
그룹 3 (192~255)
→ 픽셀값 255
256가지 다른 값을 딱 4가지로 모아버리는 게 양자화예요! 비슷한 밝기는 같은 값으로 통일됩니다.
코드로 보는 3단계 계산
1
그룹 크기 계산: group_size = 256 // levels
4단계이면 256 ÷ 4 = 64 (64개씩 한 그룹)
2
픽셀을 그룹에 배정 후 0~255로 다시 늘리기:
(img // group_size) * (255 // (levels-1))
3
데이터 타입 변환: .astype(np.uint8)
(이미지 픽셀은 반드시 정수 0~255이어야 함)
levels = 4 group_size = 256 // levels # = 64 quantized = (img // group_size) * (255 // (levels - 1)) quantized = quantized.astype(np.uint8)
단계가 줄수록 → 파일 용량은 줄지만, 이미지 품질(디테일)도 함께 줄어요. 계단 현상(banding)이 심해집니다!
